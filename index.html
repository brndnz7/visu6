Aucun indice utilisé.
Solution commentée 
function applyHeightMap(path, material) {
  const image = new Image();
  image.onload = () => {
    const canvas = document.createElement('canvas');
    canvas.width = image.width;
    canvas.height = image.height;
    const context = canvas.getContext('2d');
    context.drawImage(image, 0, 0);

    const heightData = context.getImageData(0, 0, image.width, image.height);
    material.displacementMap = new THREE.CanvasTexture(canvas);
    material.displacementScale = 0.55;
    material.needsUpdate = true;
  };
  image.src = path;
}

Base commune

Le squelette Three.js à connaître

Tous les TP suivent la même colonne vertébrale : renderer WebGL, caméra perspective, scène, objets, contrôles, boucle de rendu.

Renderer

WebGLRenderer

Il dessine la scène dans un canvas. Dans les TP : antialias: true, ajout du canvas dans #container, puis renderer.render(scene, camera).

  • setSize(width, height) définit la résolution d'affichage.
  • setClearColor(color, alpha) définit le fond si pas de background.
  • Au resize, il faut redimensionner le renderer et mettre à jour la caméra.
Scene

Scene et graphe

La scène contient des Object3D : meshes, lumières, helpers, groupes, modèles chargés. Un objet peut avoir des enfants, comme les capitales ajoutées au mesh earth.

  • scene.add(object) ajoute au monde.
  • earth.add(capSphere) attache au repère local du globe.
  • Les transformations parentales affectent les enfants.
Camera

PerspectiveCamera

Elle simule une caméra réelle avec perspective. Les paramètres clés sont fov, aspect, near, far.

  • position.set(x, y, z) place la caméra.
  • lookAt(0, 0, 0) l'oriente vers une cible.
  • near/far trop serrés peuvent couper les objets.
Boucle

Rendu continu

La boucle render() met à jour les contrôles, anime les objets, lance le raycasting, dessine la scène puis rappelle requestAnimationFrame.

  • Animation stable : modifier peu de choses à chaque frame.
  • Navigation FPS : multiplier les déplacements par delta.
  • Les stats FPS du TP1 servent à surveiller la performance.

Objets 3D

Géométries, matériaux et transformations

La formule à retenir

Mesh = Geometry + Material

La géométrie décrit les sommets et faces. Le matériau décrit comment la surface réagit à la lumière, aux textures et au rendu. Le mesh est l'objet placé dans la scène.

  • SphereGeometry(radius, widthSegments, heightSegments) pour le globe.
  • PlaneGeometry(width, height, widthSegments, heightSegments) pour le terrain.
  • Plus il y a de subdivisions, plus le displacement peut créer du relief.
  • rotateX(-Math.PI / 2) couche un plan vertical pour en faire un sol.
  • position, rotation, scale sont dans le repère du parent.
MeshBasicMaterial Ne réagit pas à la lumière. Utile pour des marqueurs, labels 3D, debug.
MeshPhongMaterial Réagit aux lumières, accepte specular, shininess, normalMap, displacementMap.
MeshLambertMaterial Éclairage diffus, souvent utilisé pour surligner un objet sélectionné.

Textures

Cartes d'image : couleur, relief, brillance

Le TP globe et le TP terrain montrent les cartes essentielles. Une texture peut venir d'un fichier image ou d'un canvas généré en JavaScript.

map

Couleur de base de la surface. Sur le globe, c'est l'image visible des continents et océans.

normalMap

Modifie les normales pour simuler de petits reliefs sans déplacer les sommets.

specularMap

Contrôle où apparaissent les reflets. Dans le TP globe, les océans brillent plus que les continents.

displacementMap

Déplace réellement les sommets. Il faut une géométrie suffisamment subdivisée.

Éclairage

Choisir la bonne lumière

Lumière Utilisation dans les TP À retenir
AmbientLight Éclairage minimal du globe. Éclaire partout pareil, ne donne pas de direction ni de relief.
DirectionalLight Soleil du globe, lumière du terrain. Ses rayons sont parallèles. Sa position représente surtout une direction.
HemisphereLight Ambiance de l'appartement et du terrain. Combine couleur du ciel et couleur du sol, pratique pour une scène lisible.

Navigation

Caméras et contrôles

OrbitControls

Utilisé dans les TP globe et terrain. La caméra orbite autour d'une cible. C'est idéal pour inspecter un objet central.

  • Appeler controls.update() dans la boucle.
  • Associer les contrôles au renderer.domElement.
  • Très pratique avec AxesHelper et GridHelper.

MyFirstPersonControls

Utilisé dans le TP environnement. La caméra se comporte comme un visiteur dans l'appartement : ZQSD/WASD ou flèches, souris maintenue pour regarder.

  • Le déplacement se fait dans le repère local de la caméra avec translateZ et translateX.
  • delta rend la vitesse indépendante du nombre de FPS.
  • La hauteur de regard est forcée avec camera.position.y = lookHeight.

Interaction

Raycasting, souris et sélection

Le raycasting transforme une position 2D de souris en rayon 3D lancé depuis la caméra. On récupère ensuite les objets intersectés.

1Mouse eventclientX/clientY
2NDC[-1, 1]
3RaycastersetFromCamera
4IntersectionsintersectObjects
5Réactioncouleur, label, sélection

Formule NDC

mouse.x = (event.clientX / window.innerWidth) * 2 - 1;
mouse.y = -(event.clientY / window.innerHeight) * 2 + 1;

Si le canvas ne prend pas toute la fenêtre, il faut utiliser le rectangle du canvas avec getBoundingClientRect().

Deux usages vus

  • TP1 : survol des capitales, changement de couleur, bulle d'info, pause de l'animation.
  • TP2 : clic droit sur un mesh de l'appartement, remplacement temporaire du matériau.
  • Extension possible : collision FPS avec un rayon lancé dans la direction du mouvement.

Données externes

Charger fichiers, CSV et modèles

Textures

TextureLoader

Charge une image et produit une texture. Le callback onLoad sert à relancer un rendu ou à déclencher la suite.

CSV

fetch() + parsing

Le TP globe charge les capitales en CSV, ignore la première ligne, convertit latitude/longitude en coordonnées 3D puis crée une sphère par capitale.

Modèle

ColladaLoader

Le TP environnement charge model.dae, ajoute collada.scene à la scène, puis parcourt le graphe pour récupérer tous les meshes.

Canvas

CanvasTexture

Le TP terrain transforme une heightmap en texture de déplacement, en normal map et en carte de couleur calculée par gradient.

Terrain

Heightmap, normal map et gradient d'altitude

Principe

Une heightmap est une image en niveaux de gris. Chaque pixel représente une altitude : noir bas, blanc haut. Three.js utilise cette image comme displacementMap pour déplacer les sommets du plan.

  1. Créer un PlaneGeometry avec beaucoup de subdivisions.
  2. Charger la heightmap dans un Image.
  3. Dessiner l'image dans un canvas pour lire les pixels.
  4. Créer une CanvasTexture et l'assigner à displacementMap.
  5. Calculer une normal map avec Sobel pour améliorer l'éclairage.
  6. Créer une carte de couleurs à partir d'un gradient d'altitude.

Normal map Sobel

Le filtre compare les voisins d'un pixel pour estimer la pente : dx horizontal, dy vertical. Le vecteur (-dx, -dy, z) est normalisé puis converti en couleur RGB. 

normalData.r = 0.5 * (normal.x + 1) * 255;
normalData.g = 0.5 * (normal.y + 1) * 255;
normalData.b = 0.5 * (normal.z + 1) * 255;

Latitude/longitude vers sphère

function latLonToVector3(lat, lon, radius) {
  const phi = lat * Math.PI / 180;
  const theta = (lon - 180) * Math.PI / 180;
  return new THREE.Vector3(
    -radius * Math.cos(phi) * Math.cos(theta),
     radius * Math.sin(phi),
     radius * Math.cos(phi) * Math.sin(theta)
  );
}

Auto-test

Questions flash avant le TP

Pourquoi faut-il appeler camera.updateProjectionMatrix() après un resize ?

Parce que la matrice de projection dépend de aspect. Sans mise à jour, l'image peut être étirée ou incohérente.

Différence entre normalMap et displacementMap ?

normalMap modifie l'éclairage apparent sans déplacer les sommets. displacementMap déplace réellement les sommets de la géométrie.

Pourquoi le TP terrain utilise un plan très subdivisé ?

Le déplacement par heightmap agit sur les sommets. Sans assez de sommets, le relief ne peut pas être détaillé.

À quoi sert raycaster.intersectObjects(objects, true) ?

À tester un rayon contre une liste d'objets. Le paramètre true inclut récursivement les enfants.

Pourquoi utiliser delta dans les contrôles FPS ?

Pour que la distance parcourue dépende du temps réel écoulé, pas du nombre de frames par seconde.

Quand utiliser MeshBasicMaterial ?

Quand l'objet doit rester visible sans lumière, par exemple des marqueurs de capitales ou des objets de debug.
